你在 Linear Probe 上能达到 92.29% 的准确率，说明你的 Encoder 其实已经提取出了**线性可分**的特征（即可以在高维空间里画几条线把不同的衣服切开）。但它并没有形成**高斯聚集**（即同类抱团）。

在对比学习中，“能分开”和“聚得紧”是两回事。要让特征在 t-SNE 上呈现出界限分明、类内紧凑的“一簇一簇”的效果，你需要对特征空间的几何分布进行强制约束。

以下是实现高阶分簇显示的完整落地方案：

### 1. 核心破局点：调低温度系数 (Temperature $\tau$)

在使用 SimCLR 或 MoCo 的 InfoNCE 损失函数时，温度系数 $\tau$ 是决定分簇紧密度的最关键参数。

* **现象**：默认的 $\tau$（通常是 0.5 或 0.07）可能对 FashionMNIST 不够敏感。
* **原理**：较低的 $\tau$ 会成倍放大“困难负样本（Hard Negatives）”的梯度惩罚。比如一件“T恤”和一件“套衫”长得很像，模型很容易把它们混在一起。降低 $\tau$ 会强迫模型必须把这对极其相似的异类狠狠推开，同时把同类死死拉近。
* **操作**：将 $\tau$ 逐步调低，比如测试 `0.1`, `0.05`, 甚至 `0.01`。你会发现虽然 Linear Probe 的准确率可能只提升了一点点甚至没变，但 t-SNE 的可视化会产生质的飞跃，类内会变得极其紧凑。

### 2. 对齐特征空间：使用非线性 Projection Head

你在 t-SNE 时，**必须使用 Encoder 的直接输出 $h$，而不是降维后的 $z$**。为了确保 $h$ 具备良好的聚类属性，Projection Head 的设计至关重要。

* **操作**：不要只用一层线性层映射。使用至少包含一层隐藏层和 ReLU 激活函数的 MLP（多层感知机）。
* **架构**：`Encoder -> h (512维) -> Linear -> BatchNorm -> ReLU -> Linear -> z (128维)`
* **原因**：对比学习的 Loss 是在 $z$ 空间计算的。非线性的 MLP 像一个缓冲区，吸收了数据增强带来的剧烈形变信息，从而使得上一层 $h$ 能够保留更多原始图像的语义信息，更利于展现出物理意义上的“簇”。

### 3. 数据增强的“克制与精准”

如前所述，由于 FashionMNIST 是 28x28 的低分辨率灰度图，盲目照搬 ImageNet 的强增强（如大比例裁剪、颜色扭曲）会破坏图像原有的拓扑结构，导致特征空间混乱。

* **操作**：
* **禁用**：`ColorJitter`（颜色抖动）、`RandomGrayscale`。
* **慎用**：`RandomResizedCrop` 的 `scale` 下限不要低于 `0.6`。如果剪裁得只剩一个袖口，模型根本不知道它是T恤还是外套，这会产生大量的“假正样本（False Positives）”，把本该分开的簇强行拉在一起。
* **推荐**：加入轻微的高斯模糊（模拟拍摄失焦）和随机遮挡（Random Erasing，模拟衣物折叠或部分遮挡），迫使模型学习衣物的全局结构轮廓。



### 4. t-SNE 算法本身的超参数调优

有时候模型已经学得很好了，是降维算法没画好。

* **困惑度 (Perplexity)**：默认通常是 30。对于 FashionMNIST 这种密集分布的数据集，将 `perplexity` 调大（例如 `50` 到 `100`），可以让 t-SNE 在降维时更多地考虑全局结构，簇与簇之间的距离会拉得更开。
* **初始化**：使用 PCA 初始化（`init='pca'`）而不是随机初始化，这样画出来的簇位置更稳定，也更反映真实的全局距离。
